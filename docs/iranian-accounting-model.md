# مدل حسابداری ایرانی

## مبنا

سیستم فقط بر مبنای تعهدی کار می‌کند. درآمد، هزینه، بدهی و مطالبه در زمان ایجاد رویداد اقتصادی ثبت می‌شوند؛ دریافت و پرداخت وجه رویداد جداگانه است.

## سلسله‌مراتب حساب

1. گروه
2. کل
3. معین
4. تفصیلی شناور

سه سطح اول روی `Account` در ERPNext نگاشت می‌شوند. تفصیلی شناور در DocType اختصاصی `ASOUD Floating Detail` نگهداری و با `ASOUD Account Mapping` به حساب‌های مجاز متصل می‌شود.

## قواعد کدگذاری

- کد گروه باید یکتا باشد.
- کد فرزند با پیشوند والد شروع می‌شود.
- طول کد تفصیلی از تنظیمات ASOUD خوانده می‌شود.
- تولید خودکار پیش‌فرض است؛ ورود دستی فقط با مجوز مناسب ممکن است.
- کد استفاده‌شده حذف فیزیکی نمی‌شود؛ رکورد غیرفعال می‌گردد.

## الگوی خودکار همه سطوح

مدیر سیستم فقط یک‌بار تعداد ارقام هر سطح را تعیین می‌کند. الگوی پیش‌فرض:

| سطح | تعداد رقم افزوده‌شده | نمونه |
|---|---:|---|
| گروه | 1 | `1` دارایی‌ها |
| کل | 2 | `101` دارایی‌های جاری |
| معین | 2 | `10101` موجودی نقد و بانک |
| تفصیلی | 5 | `1000000001` مشتری اول |

برای هر سطح، کد والد به‌عنوان پیشوند استفاده و اولین شماره آزاد تولید می‌شود. کد دستی فقط با خاموش‌کردن تولید خودکار و داشتن مجوز مدیر مالی امکان‌پذیر است.

## گروه‌های پیشنهادی تفصیلی

| کد | عنوان |
|---|---|
| 10000 | مشتریان |
| 20000 | تأمین‌کنندگان |
| 30000 | پرسنل |
| 40000 | بانک‌ها |
| 50000 | صندوق‌ها |
| 60000 | مراکز هزینه |
| 70000 | پروژه‌ها |

جزئیات قابل ویرایش یا غیرفعال‌سازی هستند؛ حذف رکورد دارای گردش مالی مجاز نیست.
# اشخاص و تفصیلی شناور

هر شخص حقیقی یا حقوقی یک پروفایل واحد دارد و می‌تواند هم‌زمان مشتری،
تأمین‌کننده، پرسنل، سهام‌دار یا سایر باشد. نقش‌ها نباید باعث ایجاد چند پروفایل
تکراری برای یک کد یا شناسه ملی شوند.

تفصیلی شناور مستقل از درخت گروه، کل و معین نگهداری می‌شود. حساب‌های معین از
طریق نگاشت کنترل‌شده به یک یا چند گروه تفصیلی متصل می‌شوند؛ بنابراین یک شخص
می‌تواند بدون تکثیر کد در چند حساب معین مجاز استفاده شود.
